1. Foundations: OAuth 2.0 vs OpenID Connect (OIDC)

Before integrating Fayda, it is crucial to understand the protocols it uses. Fayda's eSignet acts as an Identity Provider (IdP) using OpenID Connect, which is a layer built on top of OAuth 2.0.

🔑 OAuth 2.0 (Authorization)

Designed to grant a third-party application (donorLink) limited access to an HTTP service. It answers: "What is donorLink allowed to do?" It does not tell you who the user is.

🛡️ OpenID Connect (Authentication)

Built on OAuth 2.0, OIDC adds an identity layer. It introduces the id_token (a JSON Web Token) and a standardized /userinfo endpoint. It answers: "Who is this user?" This is what Fayda uses.
